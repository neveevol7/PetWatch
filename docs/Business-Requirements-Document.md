# 业务需求说明书 (BRD)：Apple Watch 虚拟宠物 (MVP)

## 1. 项目背景与目标
本项目旨在开发一款 Apple Watch 原生电子宠物应用，通过将孩子的运动数据（HealthKit）与学习任务（家长端验证）相结合，利用高品质像素风格（HD Pixel Art）和 RPG 养成机制，提升孩子的运动积极性与学习自觉性。

## 2. 核心业务流程

### 2.1 宠物生命周期管理
*   **领养：** 用户首次进入 App 领取初始宠物。
*   **生存：** 宠物拥有饥饿值、心情值、精力值，随时间自动衰减。
*   **成长：** 通过积累能量提升等级（LV）并进化。
*   **衰减/失败：** 能量归零时进入“饥饿状态”，停止收益，需特定任务复活。

### 2.2 能量获取逻辑
*   **运动（自动）：** 步数 -> 补充饥饿值。
*   **学习（家长确认）：** 完成预设任务 -> 家长手机端确认 -> 手表端增加饥饿值 + 金币。
*   **精力（无感）：** 手表进入充电状态 -> 模拟“宠物睡觉” -> 恢复精力值。

### 2.3 奖励与消费循环
*   **产出：** 宠物处于健康状态时持续产出金币。
*   **消费：** 金币用于在商店购买像素风格饰品（帽子、眼镜）及环境背景图。
*   **荣誉：** 战力值 = 等级 + 资产价值。

## 3. 功能需求清单 (Functional Requirements)

### 3.1 WatchOS 客户端 (核心)
- [ ] **HUD 主界面：** HD 像素风格宠物显示、三维属性条（饥饿、心情、精力）。
- [ ] **环境系统：** 支持分层背景图显示与动态切换。
- [ ] **任务中心：** 显示家长分配的任务，支持“点击完成”操作。
- [ ] **宠物商店：** 像素饰品预览与金币购买逻辑。
- [ ] **排行榜：** 基于 CloudKit 的全球/好友战力排名显示。

### 3.2 管理端/后台逻辑 (Backend)
- [ ] **数据持久化：** 使用 SwiftData (CloudKit 同步) 存储宠物状态、背包数据。
- [ ] **同步引擎：** 实现 iPhone (家长端) 与 Watch (孩子端) 的低延迟任务状态同步。
- [ ] **权限管理：** HealthKit 运动与睡眠权限申请。

## 4. 非功能需求
*   **视觉风格：** HD Pixel Art (16-bit/32-bit 风格，参考宝可梦)。
*   **交互：** 适配 Digital Crown (数码表冠) 滚动与触感反馈 (Haptic)。
*   **性能：** 极低功耗运行，避免影响手表日常续航。

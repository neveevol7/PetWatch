# 产品设计文档：Apple Watch 虚拟宠物 (MVP) - 最终修订版

## 1. 产品愿景
通过 Apple Watch 上的虚拟宠物养成，将孩子的运动数据与学习任务结合，利用 RPG 机制和家长互动，培养孩子的责任感与自我驱动力。

## 2. 三维属性系统 (Attribute System)

### 2.1 饥饿值 (Hunger) - 生存底线
*   **对应行为：** 运动 (HealthKit 自动读取) + 学习任务 (家长在 iPhone 核查)。
*   **衰减逻辑：** 随时间每小时自然流失。
*   **失败状态：** 降为 0 时宠物进入“饥饿变灰”状态，产金币停止。

### 2.2 心情值 (Mood) - 激励乘数
*   **对应行为：** 手表端的点击/抚摸互动。
*   **作用：** 直接影响金币产出速度。
    *   *高心情 (Mood > 80)：* 金币产出 1.5x。
    *   *低心情 (Mood < 20)：* 金币产出 0.5x。

### 2.3 精力值 (Stamina) - 规则约束 (方案 A：充电即睡眠)
*   **对应行为：** 监测 Apple Watch 的充电状态。
*   **逻辑：** 
    *   当手表处于充电状态时，宠物自动进入“休息/午睡”模式，精力值缓慢恢复。
    *   为了次日有饱满的精力，孩子需要养成每晚/定期充电的习惯。
*   **作用：** 精力值决定了每日可领取的“学习任务”上限。如果精力不足，宠物会表现为“打瞌睡”，限制高级交互。

## 3. 荣誉与奖励 (Rewards)

### 3.1 金币商店 (Coin Store)
*   **内容：** 宠物饰品（帽子、眼镜）、宠物窝/场景背景。
*   **作用：** 孩子赚取金币后，可自主打扮宠物和装点宠物的“家”。

### 3.2 宠物战力榜 (Leaderboard) (方案 D：收藏家等级)
*   **计算公式：** 战力 = (当前等级 * 100) + (已解锁饰品总价值 * 0.5) + (已解锁场景价值)。
*   **稳定性：** 战力是一个稳定增长或台阶式跳跃的数值，不随每日状态（饥饿、心情）波动。
*   **目的：** 鼓励孩子通过努力学习运动来赚金币，通过“消费”和“升级”来提升全网/好友排名。

## 4. UI/UX 布局 (WatchOS)
*   **中心区：** 2D 宠物形象（位于已购买的**背景场景**中，佩戴已购买的**饰品**）。
*   **侧边栏：** 三色小进度条（绿-饥饿、红-心情、蓝-精力）。
*   **交互：** 
    *   抬手即见主界面。
    *   下滑：任务列表。
    *   点击宠物或背景：触发互动/进入商店。

## 5. 技术架构
*   **核心：** SwiftUI + HealthKit + WatchConnectivity。
*   **同步：** CloudKit 确保家长与孩子端的数据一致性。
*   **设备：** 针对 Apple Watch 续航优化，优先使用系统内置的充电状态 API。

## 6. MVP 开发重点
1.  **充电状态检测：** 实现精力值随充电恢复的闭环。
2.  **饰品与场景系统：** 建立基础的饰品仓库和场景切换逻辑。
3.  **战力计算引擎：** 实现基于等级与资产的排名数据上报。
